# 責任與 AI 協作矩陣

本案例採 AI 協作開發。重點不是宣稱每行程式都由人工手寫，而是清楚標示問題定義、工程責任、AI 協助與驗證方式。

| 工作項目 | 作品擁有者責任 | AI 協助 | 驗證方式 |
|---|---|---|---|
| 問題與範圍 | 選定提案評審情境、角色、限制與公開邊界 | 比較案例切法與文案 | Fixture 與案例文件一致 |
| 工作流設計 | 決定角色責任、Checkpoint、Hash 與失敗收斂 | 產生設計候選與實作草稿 | 架構審查、型別檢查、測試 |
| 程式實作 | 對公開結果、維護與安全負責 | 協助建立 TypeScript 程式與測試 | `npm run check`、人工檢視 |
| 結構化輸出 | 定義必要欄位與拒絕條件 | 建議 Schema 與邊界案例 | 合法／非法輸出測試 |
| Retry／Resume | 定義可重試錯誤、Hash 守門與 Checkpoint 行為 | 協助實作與找反例 | Retry、續跑、Hash 漂移測試 |
| 公開安全 | 決定不得公開的資料與權限 | 協助敏感字串掃描 | Git Diff、Secret Scan、人工覆核 |
| 文件與展示 | 決定招聘者需要看到的證據 | 協助資訊架構與文字整理 | 連結、CI、手機與桌機檢查 |

## 私人正式平台

私人平台與公開案例不是同一份程式碼。公開案例只重新實作可獨立解釋、可重現、無私人資料的工程切片。正式平台的完整原始碼、資料庫、紀錄及權限能力不在本 Repository。
